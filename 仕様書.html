このシステムで何が変わるか

Before（今まで）After（このシステム導入後）
投稿したいけど文章を考える時間も気力もない → 結果的に投稿しない 通知が来る → 文案が3パターン届いている → 選んでコピペするだけ → Xに投稿

役割分担

やること誰が時間
投稿カレンダーに基づき3日前に通知を送るBot（自動）—
「GO」または変更内容をチャンネルに書く担当者（自分）30秒
当日朝、投稿文3パターンを自動生成して送信Claude（自動）約15秒
好きな文案をコピーしてXに投稿担当者（自分）1分
文案を修正したい場合はチャンネルに指示を書く担当者（自分）→ Claude1〜2分
担当者の作業は「返信30秒 + コピペ1分」のみ。ネタを考える必要はない。

チャンネル構成（v2：1チャンネルに統合）

チャンネル名ID用途公開範囲
sns-bot1507312995041280070通知受信・文案確認・修正依頼・突発ネタ投稿、すべてここ担当者＋Botのみ（プライベート）
チャンネルは1つだけ。担当者とBotのプライベートチャンネル。他のメンバーには見えない。

通常の投稿フロー（計画投稿）

【3日前・朝9時】自動通知
Bot が @担当者 にメンション（sns-botチャンネル）
「📅 3日後の投稿予定：[日付]　テーマ：[テーマ]
　そのまま進める場合は「GO」と書いてください。
　変更・追加情報があればそのまま書いてください。」

選手へのヒアリングが必要なテーマの場合は、通知に追加で指示が入る：
「💬 事前ヒアリング：〇〇について各選手に感想を聞いてください。」
（例：新キャラ・イングリットの投稿なら「各選手にイングリットの感想を聞いてください」と通知される）

【3日前〜前日】担当者が返信（3パターンから選べる）
　① 「GO」と書く → カレンダー通りのテーマで当日文案を生成
　② 「オニキ：使いやすい。サトル：強い。あしゅまる：対空が面白い」→ 選手コメントをネタに文案生成
　③ 「今日弟がトパチャン勝ったのでそっちで」→ 新ネタで文案生成
　→ 何を書いても必ず3パターンが返ってくる

【当日・朝8時】自動で文案を送信
Bot が sns-bot チャンネルに3パターンの投稿文を送付 ＋ @担当者 メンション

【当日】担当者が投稿
好きな文案をコピーしてXに貼り付けて投稿

【修正したい場合】sns-botチャンネルに指示を書く
「もっとシンプルに」「サトルのキャラに寄せて」など
→ 必ず3パターンが返ってくる（スマホからでも使える）
→ 気に入ったものをコピーしてXに投稿

突発・速報投稿フロー

カレンダーにない急なネタが出た時も sns-botチャンネル を使う。

① sns-botチャンネルにネタを一言書く
　例：「試合結果出た」「選手が練習で面白いこと言ってた」

② Botが 👀 リアクションで受信確認

③ 即座に3パターンの文案が届く

④ 好きな文案をコピーしてXに投稿

⑤ 修正したい場合はチャンネルに指示を書く → 3パターンが返ってくる
1つのチャンネルで全操作が完結。スマホからでも操作可能。
担当者が何を書いても → Botは必ず3パターンを返す。1対3を厳守。

カレンダー管理について

  • 半年分のスケジュールをあらかじめシステムに登録しておく
  • 残り1ヶ月を切ったら、Botが自動で警告を通知する
    　「⚠️ 投稿カレンダーの残りが1ヶ月を切りました。次の半年分のスケジュールを作成してください。」
  • 通知を受けたら次の半年分のカレンダーをシステムに追加登録する

Botの起動方法

# プロジェクトフォルダで実行
npm start

# 本番運用（サーバー再起動後も自動で立ち上がる）
pm2 start index.js --name northeption-bot
pm2 save

起動確認：以下がターミナルに表示されれば正常

✅ NORTHEPTION-bot#9789 が起動しました
📋 SNS管理チャンネル: 1507312995041280070

注意事項

  • .env ファイルは絶対にGitにコミットしない（APIキーが漏洩する）
  • Botは必ず1つだけ起動すること。複数起動すると同じ投稿に対して複数の文案が届く
  • X APIは使用していない。投稿は必ず手動でコピペして行う
  • 発信の最終判断は必ず人間が行う（全自動投稿はしない）
